Where this came from

它从哪来

On April 17, 2026, Anthropic launched Claude Design — an AI tool that reads your codebase and brand, and generates a full design system as a browseable folder. No TSX. No framework. Just tokens.css, components.css, and a set of HTML proof pages.

2026 年 4 月 17 日，Anthropic 发布了 Claude Design——一个读你 codebase 和品牌资料、反向生成完整设计系统的 AI 工具。输出是一个可浏览的文件夹：没有 TSX、没有框架，就是 tokens.css、components.css，加一组 HTML 证明页。

Staring at that folder, one thing became obvious: the "design system" a Claude Design project ships is not a React component library. It's a pure-CSS truth source with a documentation site wrapped around it. This repo packages that idea as a reusable, distributable agent skill.

盯着那份文件夹看，有一件事忽然清楚：Claude Design 输出的"设计系统"不是一个 React 组件库，而是一份纯 CSS 真相源加一个包裹它的文档站。这个仓库把这个想法打包成了可复用、可分发的 agent skill。

The architecture in one sentence

一句话讲架构

One truth source, many downstream consumers.

一份真相源，多个下游消费者。

The DNA is assets/tokens.css — every color, radius, spacing, and shadow decision lives there. components.css projects those tokens into semantic classes (.button, .card). Everything else in this repo is a consumer of that truth: the SKILL for AI agents, this docs site for humans, and any future React/Vue/email-template layer you'd add downstream.

DNA 是 assets/tokens.css——所有颜色、圆角、间距、阴影决策都在那里。components.css 把这些 token 投影成语义类（.button、.card）。仓库里其他一切都是这份真相源的下游消费者：给 AI 的 SKILL、给人的这个文档站，以及未来可以加的 React / Vue / 邮件模板层。

Change --accent once, and every page, every component, every downstream consumer follows. That's not a convenience — it's the architecture itself.

改一次 --accent，每个页面、每个组件、每个下游消费者都跟着变。这不是"便利"——这就是架构本身。

01
AI reads CSS best
AI 最擅长读 CSS
Semantic HTML + CSS variables are the densest, least-ambiguous language in Claude's training data. No cva() indirection to unwind.
语义 HTML + CSS 变量是 Claude 训练语料里最稠密、最无歧义的语言。没有 cva() 这层中介要 AI 翻译。
02
Framework independence
框架独立
Two CSS files work with React, Vue, Svelte, Astro, plain HTML, even email templates. TSX locks you to one runtime.
两个 CSS 文件可以用在 React、Vue、Svelte、Astro、纯 HTML，甚至邮件模板里。TSX 把你锁死在一个 runtime。
03
Designers can read it
设计师读得懂
Open tokens.css, see every decision. No Storybook build, no TypeScript to parse. Visual proofs live as browseable HTML.
打开 tokens.css 就能看所有决策。不用跑 Storybook、不用解析 TypeScript。视觉证明就是可浏览的 HTML。
04
Restraint by design
刻意克制
No runtime JS, no ZiProvider, no npm dependency graph. The less the skill ships, the more places it fits.
没有 runtime JS，没有 ZiProvider，没有 npm 依赖图。skill 装得越少，它能去的地方就越多。
05
Outlives frameworks
比框架活得久
Three years from now, React 19 may be legacy. Your tokens.css still runs. CSS and HTML are the Web's bedrock.
三年后 React 19 可能已成遗产，你的 tokens.css 照样跑。CSS 和 HTML 是 Web 的基岩。
